# Gouvernance

## Vue d’ensemble

Sequoïa est un projet open source publié sous licence GNU General Public
License v3.0 (GPL-3.0).  
Sa gouvernance repose sur un modèle piloté par des mainteneurs, conçu
pour garantir la cohérence technique, la pérennité du projet et
l’ouverture aux contributions.

L’objectif de cette gouvernance est de concilier :

- une vision claire et cohérente du projet,
- l’ouverture aux contributions de la communauté,
- une souveraineté partagée rendue possible par la licence GPL.

## Mainteneurs

Les mainteneurs sont responsables de l’orientation générale, de
l’intégrité et de la qualité du projet.

Leurs responsabilités incluent notamment :

- définir et orienter la feuille de route du projet,
- examiner et intégrer les pull requests,
- garantir la qualité et la cohérence du code,
- gérer les publications de nouvelles versions,
- maintenir la documentation,
- faciliter les contributions de la communauté.

Les mainteneurs agissent dans l’intérêt du projet et de ses
utilisateurs.

## Contributeurs

Toute personne peut contribuer à Rsequoia2, notamment par :

- le signalement de bugs,
- des propositions de nouvelles fonctionnalités,
- l’amélioration de la documentation,
- des contributions de code,
- des tests et retours d’expérience.

Les contributeurs participent à l’évolution du projet par leurs
propositions et leur collaboration avec les mainteneurs.

## Processus de décision

Les décisions sont prises de manière transparente via les issues et les
pull requests GitHub.

Les principes généraux sont les suivants :

- les discussions techniques sont publiques,
- les propositions sont évaluées en fonction de leur pertinence
  technique et de leur cohérence avec les objectifs du projet,
- les mainteneurs prennent les décisions finales lorsque nécessaire afin
  de garantir la cohérence du projet.

La recherche du consensus est privilégiée dans la mesure du possible.

## Open Source et souveraineté partagée

Rsequoia2 est publiés sous licence GPL-3.0, ce qui garantit que :

- le code source reste ouvert et accessible,
- chacun peut utiliser, étudier, modifier et redistribuer le logiciel,
- les améliorations restent accessibles à la communauté.

Ce modèle garantit la pérennité du projet et une souveraineté partagée
sur l’outil.

## Devenir mainteneur

Les contributeurs qui démontrent des contributions régulières, de
qualité, et alignées avec les objectifs du projet peuvent être invités à
devenir mainteneurs.

Cette décision est prise collectivement par les mainteneurs existants.

## Transparence

L’ensemble du développement est réalisé publiquement sur GitHub :

- Issues : signalement de bugs, discussions, propositions de
  fonctionnalités
- Pull Requests : propositions de modifications
- Releases : publications des nouvelles versions du logiciel

Cela garantit la transparence et la traçabilité des décisions.

## Code de conduite

Tous les participants sont invités à adopter un comportement respectueux
et constructif.

Les mainteneurs se réservent le droit de modérer les échanges afin de
préserver un environnement sain et productif.

# Governance

## Overview

Sequoïa is an open source project released under the GNU General Public
License v3.0 (GPL-3.0).  
Its governance follows a maintainer-led model, designed to ensure
technical coherence, long-term sustainability, and openness to
contributions.

The goal of this governance is to balance:

- a clear and consistent project vision,
- openness to community contributions,
- shared ownership enabled by the GPL license.

## Maintainers

Maintainers are responsible for the overall direction, integrity, and
quality of the project.

Their responsibilities include:

- defining and guiding the project roadmap,
- reviewing and merging pull requests,
- ensuring code quality and consistency,
- managing releases,
- maintaining documentation,
- facilitating community contributions.

Maintainers act in the best interest of the project and its users.

## Contributors

Anyone can contribute to Rsequoia2, including:

- bug reports,
- feature suggestions,
- documentation improvements,
- code contributions,
- testing and feedback.

Contributors help shape the evolution of the project through proposals
and collaboration with maintainers.

## Decision Process

Decisions are made transparently through GitHub issues and pull
requests.

General principles:

- technical discussions are public,
- proposals are evaluated on their technical merit and alignment with
  project goals,
- maintainers make final decisions when necessary to ensure coherence.

Consensus is encouraged whenever possible.

## Open Source and Shared Sovereignty

Rsequoia2 is licensed under GPL-3.0, ensuring that:

- the source code remains open and accessible,
- anyone can use, study, modify, and redistribute the software,
- improvements remain available to the community.

This model guarantees long-term sustainability and shared sovereignty
over the tool.

## Becoming a Maintainer

Contributors who demonstrate sustained, high-quality contributions and
alignment with the project goals may be invited to become maintainers.

This decision is made collectively by existing maintainers.

## Transparency

All development occurs publicly via GitHub:

- Issues: bug reports, discussions, feature requests
- Pull Requests: proposed changes
- Releases: versioned software updates

This ensures transparency and traceability of decisions.

## Code of Conduct

All participants are expected to act respectfully and constructively.

Maintainers reserve the right to moderate discussions to preserve a
healthy and productive environment.
